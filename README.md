## **👨‍🏫 프로젝트 소개**

TMDB(Open Movie Database) API를 활용하여

영화 정보를 탐색하고, 관심 콘텐츠를 저장할 수 있는

**React 기반 OTT 서비스 클론 프로젝트**입니다.

단순한 UI 구현을 넘어서,

**상태 관리 구조 설계**, **컴포넌트 분리**, **API 데이터 흐름 이해**를 목표로

1인 개발로 진행했습니다.

---

## **🎯 프로젝트 개요**

- **프로젝트 형태**: 개인 프로젝트 (1인 개발)
- **기술 스택**
    - React
    - React Router
    - Axios
    - Swiper
    - TMDB Open API
- **배포 환경**
    - GitHub Pages (정적 배포)
    

---

## **💡 주요 구현 내용**

- **TMDB API 연동**
    - 현재 상영작, 인기 영화, 평점 높은 영화, 개봉 예정작 등 카테고리별 데이터 호출
    - `axios.all`과 `axios.spread`를 활용한 다중 API 요청 처리
- **슬라이드 UI 구성**
    - Swiper 라이브러리를 활용해 OTT 서비스 형태의 콘텐츠 탐색 UI 구현
    - 반응형 설정을 통해 PC / 모바일 화면 대응
- **관심 목록(My List) 기능**
    - 영화/TV 콘텐츠를 즐겨찾기에 추가 및 제거
    - 여러 페이지에서 동일한 관심 목록 데이터를 공유
- **라우팅 기반 페이지 구성**
    - 메인, TV, 검색, 상세 페이지, 마이리스트 페이지 분리
    - URL 파라미터를 활용한 상세 정보 조회

## **🚀 해결한 문제 & 성과**

- **문제:**
    
    관심 목록 기능 구현 과정에서
    
    여러 페이지(Main, Detail, MyList)에서 동일한 상태를 공유해야 했고,
    
    **상태를 어디에서 관리해야 할지 판단하는 것이 가장 어려웠습니다.**
    
- **해결:**
    
    관심 목록 데이터를 전역처럼 사용해야 한다는 점을 인지하고,
    
    상태(`favlist`)와 관련 로직을 **커스텀 훅(`useFavListData`)으로 분리**하여
    
    App 컴포넌트에서 관리하도록 설계했습니다.
    
    이를 통해 상태와 UI를 분리하고, props를 통해 필요한 컴포넌트에 전달했습니다.
    
- **성과:**
    - 상태 관리 책임을 한 곳으로 모아 **데이터 흐름을 명확히 이해**
    - 관심 목록 로직의 재사용성과 유지보수성 향상
    - “상태는 어디에 두어야 하는가”에 대한 기준을 경험을 통해 체득
